## 超卖解决方案总结

1. **悲观锁**（`synchronized`/`ReentrantLock`）：

   - 核心思想：假设一定会发生并发冲突，先加锁再操作
   - 优点：实现简单，安全可靠
   - 缺点：并发性能较低，线程排队等待

2. **乐观锁**（数据库版本号 / 时间戳）：

   sql

   ```mysql
   UPDATE stock SET count = count - 1 WHERE id = 1 AND version = 1;
   ```

   - 核心思想：假设不会发生冲突，更新时验证版本号
   - 优点：并发性能高，适合读多写少场景
   - 缺点：需要数据库支持，失败需要重试

3. **分布式锁**（Redis/Zookeeper）：

   - 核心思想：跨 JVM 进程的锁机制
   - 适用场景：微服务架构、分布式系统

4. **队列化处理**：

   - 核心思想：将并发请求放入队列，串行化处理
   - 适用场景：高并发秒杀系统
