## 一、通俗理解学习（零基础看懂，不背书）

### 1、什么是垃圾？

JVM 堆内存里**没有任何线程引用、再也用不到的对象**，就是垃圾

GC 垃圾回收 = JVM 自动清理这些无用对象，释放堆内存

------

## 二、垃圾判定算法（谁是垃圾？）

### 1、引用计数法（早期方案，Java 不用）

给对象加计数器，有人引用 + 1，引用失效 - 1

计数器 = 0 → 判定为垃圾

**致命缺陷：循环引用无法回收**，内存永久泄漏，Java 彻底不用

### 2、可达性分析算法（Java 官方唯一标准）

1. 找

   GC Roots 根对象

   ：永远不会被回收的对象

   - 虚拟机栈中正在引用的对象
   - 本地方法栈 Native 引用对象
   - 静态变量、常量引用对象
   - 活跃线程引用对象

2. 从 GC Roots 往下遍历引用链

3. 遍历不到的对象 = 垃圾

   ，等待回收

   完美解决循环引用问题

------

## 三、四大引用级别（面试高频）

1. 强引用

   ```java
   Object obj = new Object()
   ```

   ​    只要有强引用，GC永远不会回收，内存满了直接抛 OOM

2. 软引用 SoftReference：内存充足不回收，内存不足 OOM 前优先回收

   场景：缓存

3. 弱引用 WeakReference：只要 GC 扫描到，立刻回收

   场景：ThreadLocal、临时缓存

4. 虚引用 PhantomReference：不影响对象生命周期，仅做回收通知

   场景：堆外内存回收管理

------

## 四、垃圾回收三大核心算法

### 1、标记 - 清除算法

流程：标记所有垃圾 → 统一清除

缺点：

- 内存碎片化严重，找不到连续大内存
- 两次遍历，效率低

### 2、标记 - 复制算法（新生代默认）

流程：内存分成两块，存活对象复制到空白块 → 清空整块旧内存

优点：无内存碎片、速度快

缺点：浪费一半内存空间

### 3、标记 - 整理算法（老年代默认）

流程：标记垃圾 → 存活对象向一端移动 → 清理边界外垃圾

优点：无内存碎片，空间利用率 100%

缺点：移动对象耗时，效率偏低

------

## 五、JVM 分代垃圾回收（核心原理）

Java 堆分为**新生代 + 老年代**，对象年龄不一样，生命周期天差地别

1. 新生代：新建对象，朝生夕死，90% 对象存活不到 1 次 GC

   用：标记 - 复制算法，MinorGC 轻快、频繁、停顿极短

2. 老年代：熬过多次新生代 GC、存活很久的大对象

   用：标记 - 整理算法，FullGC 缓慢、耗时、STW 长

补充：元空间、栈、程序计数器**不参与 GC**

只有堆内存对象需要垃圾回收

------

## 六、四大垃圾收集器（工作 + 面试必背）

### 1、Serial 串行收集器

单线程 GC，垃圾回收时所有业务线程全部暂停 STW

简单稳定，单核 CPU 老式场景

### 2、Parallel 并行收集器（JDK8 默认新生代）

多线程并行垃圾回收，主打**高吞吐量**

优先保证业务执行时间，适合后台计算、大数据场景

### 3、CMS 并发收集器（JDK8 默认老年代）

垃圾回收和业务线程**并发同时执行**，STW 停顿极短

主打低延迟，互联网响应优先场景

缺点：内存碎片、CPU 占用高、浮动垃圾问题

### 4、G1 垃圾收集器（JDK9 默认全局收集器）

- 把堆拆成一个个 Region 区块

- 新生代老年代不分家，统一回收

- 可预测停顿时间，兼顾吞吐量 + 低延迟

- 不产生内存碎片

  目前线上生产绝对主流

### 5、ZGC（JDK11+）

超低延迟，几乎毫秒级 STW，超大堆内存（TB 级）

阿里、大厂线上新一代标配

------

## 七、MinorGC 和 FullGC 区别

1. **MinorGC 新生代 GC**

- 只回收新生代
- 速度极快，停顿极短
- 频繁触发，几乎不影响业务

1. **FullGC 全堆 GC**

- 回收新生代 + 老年代

- 耗时极长，STW 长时间卡顿

- 严重影响接口 QPS、秒杀雪崩

  触发场景：老年代满了、MinorGC 后新生代放不下、元空间溢出

------

## 八、底层深度原理

1. STW Stop The World

   GC 工作时，暂停所有用户业务线程，只允许 GC 线程运行

   G1/ZGC 极致缩短 STW 时间，毫秒级

2. 分代回收本质：根据对象生命周期长短，匹配最优算法

3. 内存分配规则：大对象直接进老年代、长期存活对象晋升老年代

------

## 九、高频面试满分标准答案

### 1、Java 怎么判断对象是垃圾？

可达性分析算法，从 GC Roots 遍历引用链，不可达即为垃圾

### 2、MinorGC 和 FullGC 区别？

MinorGC 只回收新生代，快、短、频繁；FullGC 全堆回收，慢、长、极易线上故障

### 3、G1 和 CMS 区别？

- CMS 只回收老年代，G1 全堆统一回收
- CMS 有内存碎片，G1 无碎片
- G1 可控制最大停顿时间，CMS 不可控
- G1JDK9 默认，CMS 已废弃

### 4、为什么新生代用复制算法，老年代用标记整理？

- 新生代对象死亡率极高，复制算法快无碎片
- 老年代对象存活久，不能浪费一半内存，用标记整理无碎片

### 5、强软弱虚引用区别及使用场景？

按强弱顺序，回收优先级越来越高

强引用永不回收，软引用内存不足回收，弱引用 GC 就回收，虚引用做回收监听

### 6、什么是 STW？有什么影响？

GC 暂停所有业务线程，会导致接口超时、服务卡顿、秒杀雪崩

------

## 十、线上实战避坑

1. 频繁 FullGC = 线上服务雪崩、QPS 暴跌
2. 内存泄漏：对象一直有引用，永远不被 GC，慢慢堆 OOM
3. 大对象过多，直接进入老年代，频繁触发 FullGC
4. 线程池不关闭、ThreadLocal 乱用，导致内存泄漏
