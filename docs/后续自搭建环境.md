## 四、Windows 部署方案（3 选 1）

### 方案A：Docker Desktop（推荐 ⭐）





```
优点：一键启动全部服务
缺点：Docker Desktop 启动慢，吃额外 1G 内存
 
步骤：
  1. 装 Docker Desktop（开启 WSL2 后端）
  2. 写一个 docker-compose.yml 编排：
     - postgres + pgvector
     - redis
     - elasticsearch
     - kafka + zk
  3. 后端 / 前端用 IDEA / npm 直接跑（不放 Docker，便于调试）
  4. Ollama 本地装（不放 Docker，能用 GPU）
 
启动命令：
  docker compose up -d        # 启动数据库依赖
  ollama serve                # 启动本地 LLM
  IDEA 启动后端 + 前端
```

### 方案B：原生安装（更轻量）





```
优点：无 Docker 开销，性能最好
缺点：装服务繁琐
 
各服务下载链接：
  PostgreSQL    : https://www.postgresql.org/download/windows/
                  装完后用 pgxnclient 装 pgvector
  Redis         : https://github.com/tporadowski/redis/releases (Windows 版)
  Elasticsearch : https://www.elastic.co/downloads/elasticsearch (zip 解压即用)
  Kafka         : https://kafka.apache.org/downloads (有 Windows 启动脚本)
  Ollama        : https://ollama.com/download/windows (一键装)
 
Windows 服务化：用 NSSM 把每个组件注册成 Windows 服务，开机自启。
```

### 方案C：WSL2 内跑 Linux（最贴近生产）





```
优点：和云服务器环境一致，部署脚本可复用
缺点：WSL2 内 GPU 直通配置稍麻烦
 
步骤：
  1. wsl --install -d Ubuntu-22.04
  2. WSL 内装 Docker + Ollama（CUDA on WSL 现在支持了）
  3. Windows 端浏览器访问 localhost:5173 (WSL 自动端口转发)
 
适合：想模拟"上线生产"的最终演练
```

## 五、本地 LLM 接入 Spring AI（Ollama 方式）

yaml





```
# application.yaml 改 3 处即可
spring:
  ai:
    ollama:
      base-url: http://localhost:11434
      chat:
        model: qwen2.5:7b-instruct-q4_K_M
        options:
          temperature: 0.3
      embedding:
        model: bge-m3
```

bash





```
# 本地拉模型（一次性）
ollama pull qwen2.5:7b-instruct
ollama pull qwen2.5-coder:7b      # AI SQL 用
ollama pull bge-m3                 # Embedding 用
```





```
依赖切换：
  pom.xml 把 spring-ai-openai-starter
       换成 spring-ai-ollama-starter
  其他代码 0 修改（ChatClient API 完全一样）
 
测试：
  curl http://localhost:11434/api/generate -d '{
    "model": "qwen2.5:7b-instruct",
    "prompt": "Java是什么"
  }'
```
