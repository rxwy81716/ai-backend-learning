## 一、AI 项目整体架构（后端核心链路）

整体模块：

请求入口 → 参数校验 → 会话管理 → Prompt 组装 → 上下文裁剪 → 大模型调用 → 结果封装 → 会话持久化 → 异常兜底

## 二、AI 接口完整调用流程图（文字结构化，可直接手绘笔记）

```
【前端】
   ↓
1. 携带参数：sessionId / 提问内容 / 流式标识
   ↓
【Controller 接收请求】
   ↓
2. 统一参数校验、基础拦截
   ↓
【Service 业务层】
   ↓
3. 通过sessionId查询Redis
   ├─ 无历史 → 初始化System系统提示词
   └─ 有历史 → 读取完整上下文聊天记录
   ↓
4. 追加当前用户最新提问（UserMessage）
   ↓
5. 第一层防护：按消息条数限制，防止消息无限堆积
   ↓
6. 封装为 SpringAI 标准消息对象
   （SystemMessage / UserMessage / AssistantMessage）
   ↓
7. 第二层核心防护：Token精准截断
   ├─ Tokenizer 计算全部消息总Token
   ├─ 超出安全阈值
   └─ 保留系统提示 + 成对删除最早一轮问答
   ↓
8. 组装完整 Prompt 请求体
   ↓
【SpringAI 框架层】
   ↓
9. 封装HTTP/SDK请求 → 调用大模型厂商接口
   ↓
【大模型 LLM】
   ↓
10. 接收上下文 + 历史对话 + 系统指令
    ↓
11. Transformer推理 → 逐Token生成回答
    ↓
【响应回流】
   ↓
12. 同步/流式SSE 返回AI结果
   ↓
13. 拼接AI回答(AssistantMessage)，写入会话列表
   ↓
14. 更新Redis上下文，维持多轮记忆
   ↓
15. 全局异常捕获 → 超时/密钥错误/限流 兜底文案
   ↓
【前端】展示回答内容
```

## 三、横向两大模式流程图（同步 / 流式）

### 1、普通同步聊天流程

```
用户请求
   ↓
拼接完整上下文+裁剪
   ↓
chatModel.call() 阻塞调用
   ↓
大模型完整生成全部内容
   ↓
一次性返回全文
```

### 2、SSE 流式聊天流程

```
用户请求
   ↓
上下文组装+Token裁剪
   ↓
chatModel.stream() 响应式调用
   ↓
大模型逐段返回Token碎片
   ↓
MediaType:text/event-stream 长连接
   ↓
前端逐字渲染，打字机效果
```

## 四、项目核心知识点复盘（必记）

1. **大模型无状态**

   本身不存聊天记录，所有记忆 = 后端手动拼接上下文 + Redis 存储。

2. **三层约束控制输出**

- System 角色提示：规范身份、语气、答题规则
- Prompt 工程：结构化指令，减少幻觉
- 上下文裁剪：控制 Token，防止超限报错、回答失忆

1. **上下文两层防护**

- 简单兜底：限制最大消息条数
- 生产核心：Tokenizer 统计 Token + 老旧问答成对删除

1. **SpringAI 核心价值**

   统一抽象 ChatModel、Message、Prompt，

   切换豆包 / 通义 / OpenAI 只改配置，不动业务代码。

2. **线上常见故障 & 解决方案**

- 上下文过长 → Token 截断机制
- 回答乱编幻觉 → 强系统提示 + 约束 Prompt
- 接口超时 → 流式 SSE + 超时时间配置
- 多用户聊天串号 → sessionId 会话隔离
- 会话过期 → Redis 自动过期策略

## 五、项目整体技术栈复盘

- 核心框架：SpringBoot + SpringAI
- 会话存储：Redis 序列化存储多轮消息
- 流量交互：HTTP 同步接口 + SSE 流式长连接
- 上下文治理：消息条数限制 + Token 精准裁剪
- 安全控制：系统提示词约束、参数校验、全局异常处理

## 六、面试口述总结（直接背）

AI 聊天接口完整调用流程：

前端携带会话 ID 和问题请求后端，后端根据会话 ID 从 Redis 获取历史对话，拼接系统提示、历史上下文与当前问题；

先按消息条数做简单兜底，再通过 Tokenizer 计算 Token 数量，超出阈值则成对删除最早问答，完成上下文裁剪；

通过 SpringAI 组装 Prompt 调用大模型，同步一次性返回或 SSE 流式分段返回；

拿到 AI 回答后追加进会话记录，重新存入 Redis 实现多轮记忆，全局捕获异常做业务兜底。
