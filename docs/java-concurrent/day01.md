## Day1 线程基础 学习记笔记

### 核心知识点(必记)

#### 1.线程与进程的区别(理解就好)

- 进程: 一个独立的程序(比如打开的微信,IDEA),是资源分配的最小单位
- 线程: 进程内的执行单元(比如微信通同时发消息,刷朋友圈),是调度执行的最小单位,共享进程的资源

#### 2. JAVA 4种创建线程方式(重点)

1. 继承Thread类
   - 步骤:自定义类继承Thread 重写run()方法,创建对象调用start()方法
     - 优点:写法简单;
     - 缺点: java单继承,继承Thread后不能再继承其他类,耦合度高
2. 实现Runnable接口(开发首选)
   - 步骤:自定义类实现Runnable,重写run()方法,将该类对象作为参数传入Thread,调用start();
     - 优点: 解耦(线程任务和线程本身分离),可多实现,灵活度高
     - 缺点: 无返回值 不能抛异常
3. 实现Callable+Future(有返回值场景用)
   - 自定义类实现Callable<返回值类型>，重写call()方法，用FutureTask包装，传入Thread调用start()，通过get()获取返回值。
     - 优点: 有返回值,可抛出异常;
     - 缺点: 写法稍复杂
4. 线程池创建

### 3. 线程6种状态（面试高频，口述能说清即可）

1. NEW -> RUNNABLE -> BLOCKED -> WAITTING -> TIME_WAITTING -> TERMINATED
2. 新建(创建对象) - 就绪(start() 后 等待CPU调度) - 运行 (CPU执行run()) - 阻塞/等待 (遇到锁,sleep等) - 终止 (run()执行完)

### 4. 线程常用方法

- start() 开启新线程,调用底层系统方法,让线程进入就绪状态,不能重复调用
- run() 普通方法,仅执行线程任务,直接调用不会开启新线程,只是主线程执行该方法.
- sleep(long time) 让线程休眠指定时间,不释放锁,休眠结束后进入就绪状态
- join() 让调用该方法的线程先执行完,再执行其他线程.
- yield() 礼让线程,让当前线程放弃CPU执行权,重新进入就绪状态

### 5. 易错点

1.  误区：调用run()方法能开启新线程 → 错误！只有start()才能开启新线程，run()只是普通方法调用。
2. 误区：继承Thread比实现Runnable好 → 错误！Runnable更灵活，避免单继承限制，开发中优先用。
3.  重点区分：sleep() vs wait()（今日了解，面试高频）
   -  sleep()：不释放锁，休眠时间到自动唤醒，属于Thread类方法。
   - wait()：释放锁，需要notify()/notifyAll()唤醒，属于Object类方法（今日暂不深入，记区别即可）。

### 6. 总结

1. 3个Demo均能正常运行，核心是通过不同方式实现线程任务，最终都需要调用start()开启线程。
2.  Callable的get()方法会阻塞主线程，直到子线程执行完并返回结果（比如今日Demo中，主线程会等待线程3执行完，再打印返回值100）。
3. 线程命名：可以通过setName()方法，或Thread构造方法传入名称，方便调试时区分线程。

### 7. 面试必备

Java创建线程有4种方式，继承Thread耦合高、受单继承限制，实现Runnable更灵活，Callable适合需要返回值的场景，start()才是开启新线程的核心方法。
