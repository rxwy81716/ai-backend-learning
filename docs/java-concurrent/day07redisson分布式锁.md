# 面试必背 3 大核心问题

## ① 锁超时问题

业务执行时间 > Redis 锁过期时间

- 锁提前释放
- 其他线程拿到锁
- 并发重复操作 → **超卖、数据错乱**

## ② 锁续期（看门狗 WatchDog）

Redisson 默认机制：

- 每隔**10 秒**检查锁是否还被持有
- 业务没结束，自动**延长过期时间**
- 不用手动设置过期时间，彻底解决超时失效

## ③ 重复解锁问题

1. 线程 A 解锁完毕
2. Redis 锁已经过期，线程 B 拿到锁
3. 线程 A 再执行 unlock () → **误删 B 的锁**

Redisson 解决方案：

- 锁自带**线程 ID 标识**
- 只有**持有锁的当前线程**才能解锁
- 别人的锁，绝对删不掉

# 一句话面试总结

1. 原生 Redis SETNX 分布式锁：超时失效、不可续期、误删锁
2. Redisson 分布式锁：**看门狗自动续期 + 线程唯一标识防误删**
3. 完美解决集群环境库存并发超卖问题
