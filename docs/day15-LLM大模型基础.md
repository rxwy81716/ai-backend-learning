## 一. LLM大模型到底是什么

### 1. 本质

大语言模型LLM= 基于transformer神经网络架构 经过万亿级海量互联网文本训练出来的词语概率预测模型

### 2. 核心逻辑(全世界所有大模型统一原理)

AI不会思考,不会理解,不会查数据库

它只会做一件事:

根据前面所有文字,预测下一个出现概率最高的一个字/词

一字一字预测,连成整段通顺回答

### 3. 灵魂: 自注意力机制 Self-Attention

transformer核心

AI可以识别一句话里词语之间的关联关系

例如: 小明把他的狗放开了,它很开心

注意力机制让AI知道 它= 狗,不是小明

### 4. 为什么会出现幻觉

模型只学文字通顺规律,不学事实对错

没见过,不确定的内容,也会按照语法逻辑编造合理句子

真假无法自检 -> 幻觉

## 二 token详解

### 1. token是什么(中文命名 词元)

大模型最小计算单位

不是汉字,不是词语,是模型拆分文本的最小片段

- 英文: 单词/子单词
- 中文: 词组,单字,标点都会被切成Token

### 2. token换算规则

- 1汉字 ≈ **1.3 ~ 1.5 个 Token**
- 1000Token ≈ 700个汉字左右
- 标点,空格,英文,数字全部占用Token

### 3. token作用

1. 模型所有计算,训练,推理.只认Token 不认文字
2. 对话长度,收费计费,上下文限制,全部按token算
3. token越多,推理越慢,费用越高

## 三 上下文窗口长度Context Length（重点难点）

### 1. 定义

模型单次对话最大能处理的Token总数上限

常见: 8K 32K 128K 512K Token

包含内容:

- 系统提示词
- 用户历史所有回答
- AI历史所有回答
- 当前最新问题
- AI即将生成的回答

### 2.  超长上下文会发生什么

模型超过窗口上限,自动丢弃最早的对话内容

> 前面聊天失忆,逻辑错乱,回答不连贯,上下文关联失效

### 3. 多轮对话底层原理

AI本身没有长期记忆

每一次新提问,都必须把全部历史聊天记录+新问题一起重新发给模型

模型才能记住上文

### 4. 开发必须处理

Token溢出截断策略

保留最新对话,丢弃最早对话,保证不超窗口

## 四 大模型标准调用格式(REST接口完整版)

### 1. 单论问答格式

```json
{
  "model": "doubao-pro",
  "messages": [
    {"role":"user","content":"请介绍Java后端"}
  ],
  "temperature": 0.2,
  "max_tokens": 2048,
  "stream": false
}
```

### 2. 多轮对话标准格式(90%的项目用这个)

```json
{
  "model": "doubao-pro",
  "messages": [
    {"role":"system","content":"你是专业Java后端工程师，回答严谨简洁"},
    {"role":"user","content":"什么是JVM"},
    {"role":"assistant","content":"JVM是Java虚拟机，运行Java字节码..."},
    {"role":"user","content":"JVM堆栈区别是什么"}
  ],
  "temperature": 0.1,
  "max_tokens": 1024,
  "stream": true
}
```

### 3. 每个参数详细含义

1. model : 调用具体哪个大模型版本
2. system: 角色设定,规定AI身份,语气,规则
3. user: 用户提问
4. assistant: AI历史回答
5. temperature: 随机性
   - 0.1-0.3: 严谨,事实,业务回答
   - 0.7-1.0: 创意 聊天 写作
6. max_tokens: AI最多生成多少Token问答
7. stream: 是否流式输出(打字逐字返回)

## 五 完整对话流转流程

1. 用户输入文字
2. 文本 -> token分词转换
3. 拼接系统角色+历史上下文
4. 传入Transformer模型计算
5. 逐token预测下一个词
6. token返回文字
7. 返回前端(同步/流式)

## 六 0基础必备总结

1. LLM = transformer+注意力机制, 预测下一个Token生成文本
2. Token是模型最小单位,汉字 ≠Token
3. 上下文长度 = 最大Token总量,超了就丢前文,失忆
4. 多轮对话 = 每次都带上全部历史上下文
5. AI无记忆,易幻觉,所以后面要学RAG检索知识库
