## 一、通俗理解学习（自己看懂 + 不背书）

JVM 把 Java 程序运行内存，严格划分成**5 个独立区域**，每个区域分工、生命周期、OOM 风险完全不一样

### 1. 程序计数器(PC寄存器)

- 作用:记录当前线程正在执行的字节码指令地址
- 线程特点: 线程私有 每个线程独立一份
- 内存特点: JVM所有区域里唯一不会OOM内存溢出的区域
- 为什么线程私有: 多线程CPU切换上下文,靠它恢复线程执行到哪一步

### 2. 本地方法栈

- 作用: 存放本地Native方法(C/C++底层方法) 执行的局部变量
- 线程特点:线程私有
- 异常:栈深度超过JVM限制 抛出StackOverflowError

### 3. 虚拟机栈(我们常说的java栈)

- 作用:每个方法执行时,都会创建一个栈
- 存放:局部变量表,操作数栈,动态链接,方法返回地址
- 线程特点:线程私有,生命周期和线程一致
- 高频异常:
  1. 方法嵌套太深 - stackOverflowError 栈溢出
  2. 动态扩容栈内存不足 - outOfMemoryError

### 4. 堆(Heap)

- 作用: 所有线程共享,存放new出来的所有对象实例
- JVM内存最大的一块区域
- 垃圾回收 GC主要工作区域
- 异常: 对象太多堆放不下 outOfMemoryError: Java Heap space
- 特点: 线程共享,所有线程共用一块堆内存

### 5. 元空间(Metaspace JDK8+)

jdk8之前叫永久代PermGen

- 作用: 存放类信息,常量,静态变量,方法字节码,反射相关数据
- 线程特点:所有线程共享
- JDK8重大改动: 永久一处JVM堆,改用本地物理内存元空间
- 异常: 类加载太多,反射动态生成类太多 - > OutOfMemoryError: Metaspace

## 二、底层深度原理拆解

### 1、线程私有 vs 线程共享 核心区别

|    区域    | 线程归属 |    生命周期     | GC 是否回收 |
| :--------: | :------: | :-------------: | :---------: |
| 程序计数器 | 线程私有 | 随线程创建销毁  |    不 GC    |
|  虚拟机栈  | 线程私有 | 随线程创建销毁  |    不 GC    |
| 本地方法栈 | 线程私有 | 随线程创建销毁  |    不 GC    |
|     堆     | 线程共享 | 随 JVM 启动销毁 |   频繁 GC   |
|   元空间   | 线程共享 | 随 JVM 启动销毁 |   很少 GC   |

私有区域：线程结束就释放内存，完全不需要垃圾回收

共享区域：所有线程共用，必须 GC 管理，极易内存溢出

### 2、JDK7 vs JDK8 永久代 → 元空间 为什么改？

1. 永久代在堆里，和对象共用内存，容易堆 OOM
2. 永久代大小很难预估，容易溢出
3. 元空间直接用操作系统本地内存，不受 JVM 堆大小限制
4. 方便 G1 垃圾回收优化

### 3、高频底层关联知识点

1. volatile 可见性：和 JMM Java 内存模型相关，和运行时 5 大内存区域不是一个概念！
2. DCL 单例懒汉为什么加 volatile：禁止指令重排序，程序计数器指令顺序错乱问题
3. 栈内存小、速度极快；堆内存大、速度慢

## 三、高频面试满分标准答案

### 1、JVM 运行时 5 大内存区域分别是什么？各自作用？

按顺序背：程序计数器、本地方法栈、虚拟机栈、堆、元空间

分别说清作用、线程私有 / 共享、会不会 OOM

### 2、哪个区域不会内存溢出 OOM？

**程序计数器**

### 3、栈和堆的区别？

1. 栈线程私有，堆线程共享
2. 栈存放局部变量，堆存放对象实例
3. 栈自动分配释放，速度快；堆 GC 回收，速度慢
4. 栈空间小固定，堆空间大灵活

### 4、JDK8 为什么把永久代换成元空间？

1. 永久代受堆内存限制，容易 OOM
2. 元空间使用本地内存，不占用 JVM 堆
3. 优化 GC 回收，减少 FullGC
4. 动态代理、反射类不会撑爆永久代

### 5、StackOverflowError 和 OOMError 区别？

- StackOverflow：虚拟机栈深度不够，方法递归无限嵌套
- OOM：堆 / 元空间内存不够，对象太多、类太多

### 6、方法运行时，局部变量存在哪？new 对象存在哪？

- 局部变量：**虚拟机栈**
- new 对象：**堆**
- 类信息常量：**元空间**
