### 一. 什么是Prompt工程

1. 本质: 用精准,规范的指令 引导大模型稳定输出,专业 无幻觉 符合业务格式的回答

2. 大模型本身没有理解能力,你怎么提问,他就怎么生成

   烂提示 = 废话,混乱,编造,逻辑跳跃

   好提示 = 结构化,底层原理,面试级,项目可用

3. 核心价值: 大幅降低AI幻觉,统一输出格式,适配Java后端/RAG知识库场景

### 二. Prompt黄金万能公式(crispe必背)

1. C角色ROLE: 定义AI身份(java结构师,面试官,高并发专家)
2. R场景Context: 线上生产/面试/秒杀/微服务环境
3. I 要求Instruction: 长度,深度,分点,禁止幻觉
4. S思维链CoT: 强制一步步推理,不跳步
5. P示例Example: 少样本Few-shot,模仿回答格式

通用模板

> 你是 5 年 Java 高并发后端架构师
>
> 场景：互联网微服务线上高可用环境
>
> 请按【原因→现象→解决方案→优缺点】分步讲解
>
> 严禁编造技术知识点，不知道就如实说明
>
> 用 Markdown 有序列表输出，简洁专业

### 三. 同意需求 4中Prompt对比效果差异

#### 统一需求：讲解 Redis 分布式锁 秒杀防超卖

##### 1、劣质口语 Prompt（最差效果）

```
redis怎么防止秒杀超卖
```

AI 效果：

回答简短杂乱、逻辑跳跃、只讲皮毛、不提锁超时、可重入、集群锁问题、极易幻觉、无线上坑、不能面试用

##### 2、普通基础 Prompt（一般效果）

```
你是Java程序员，讲Redis分布式锁解决库存超卖
```

AI 效果：

语句通顺，但结构零散、不分优先级、不区分单机 / 分布式、不讲看门狗、不对比 Redisson 缺陷

##### 3、优质结构化 Prompt（合格线上效果）

```
你是大厂秒杀架构工程师
详细讲解Redis分布式锁解决秒杀库存超卖问题
要求：
1. 秒杀超卖并发根源
2. 原生SETNX锁致命漏洞
3. Redisson看门狗续期原理
4. 分布式锁三大经典问题
分点清晰，只讲生产实战方案，不编造内容
```

AI 效果：

条理清晰、重点齐全、无废话、无幻觉、可直接当做笔记 & 面试答案

##### 4、顶级 CoT 思维链 Prompt（最佳面试效果）

```
你是Java后端面试官，请一步步推导问题：
第一步：分析多线程并发库存竞态问题
第二步：为什么本地锁无法解决分布式超卖
第三步：SETNX+过期时间 死锁&锁提前释放漏洞
第四步：Redisson可重入+看门狗机制
第五步：Redis集群主从切换锁失效方案
严格分步推理，不跳跃逻辑，结合线上故障坑
```

AI 效果：

层层递进、底层原理透彻、架构深度拉满、零幻觉、大厂标准答案级别

### 四 两大高级Prompt技巧

#### 1. Few-shot 少样本提示

给AI1个回答范例,AI自动严格模仿格式,逻辑,风格

不用反复写字约束,输出高度统一

```plaintext
按照以下格式回答
例：问题：MQ消息丢失？
回答：1.生产者 2.Broker 3.消费者
请同格式回答：Redis缓存雪崩原因？
```

#### 2. COT思维链提示

强制AI分步思考,而不是直接给答案

复杂技术题幻觉下降90%

JVM,分布式,中间件底层必用

### 五 抑制幻觉专用固定Prompt

```plaintext
你仅基于Java官方、中间件标准技术知识回答
不确定、没见过的内容直接回复无法解答
严禁推测、杜撰、编造原理与线上方案
不虚构不存在架构与配置
```

### 六 Prompt与Temperature搭配规律

1. `0.1~0.3` + 严谨结构化 Prompt

   面试、知识库 RAG、业务问答 → 精准无幻觉

2. `0.7~1.0` + 创意 Prompt

   文案总结、逻辑梳理、聊天发散

## 七 面试必背问答

1. 为什么要学习 Prompt 工程？

统一输出格式、降低幻觉、适配私有 RAG 知识库、贴合项目业务

2. CoT 思维链有什么用？

强制分步推理，解决复杂技术逻辑错乱，大幅减少幻觉

3. Few-shot 少样本好处？

低成本规范 AI 输出格式，不用大量文字约束指令
