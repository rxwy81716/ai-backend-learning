## Redis 基础

### 一、核心知识点（String / Hash / 过期策略 / 持久化）

### 1. String 字符串（最常用）

- **结构**：key-value，value 是字符串 / 数字

- 常用命令

  ```redis
  SET key value设置指定 key 的值
  SETNX key value只有在 key 不存在时设置 key 的值
  GET key获取指定 key 的值
  MSET key1 value1 key2 value2 ……设置一个或多个指定 key 的值
  MGET key1 key2 ...获取一个或多个指定 key 的值
  STRLEN key返回 key 所储存的字符串值的长度
  INCR key将 key 中储存的数字值增一
  DECR key将 key 中储存的数字值减一
  EXISTS key判断指定 key 是否存在
  DEL key（通用）删除指定的 key
  EXPIRE key seconds（通用）给指定 key 设置过期时间
  
  ```

- **使用场景**：简单缓存、验证码、计数器、分布式锁、Session 存储

### 2. Hash 结构（对象缓存神器）

- **结构**：key → (field-value)，适合存**对象**

- 常用命令

  ```redis
  hset key field value   存字段
  hget key field         取字段
  hgetall key            取所有字段
  ```

- **场景**：用户信息、商品详情、配置信息（比 JSON 省内存、可单独修改字段）

### 3. 过期 + 淘汰策略（面试高频）

#### （1）过期策略（key 到期了怎么删）

- **定时删除**：到时间立刻删（内存友好，CPU 不友好）
- **惰性删除**：查询时才检查是否过期（CPU 友好，内存不友好）
- **定期删除**：Redis 默认：**定期 + 惰性**混合使用

#### （2）内存淘汰策略（内存满了删谁）

- `noeviction`：不删，直接报错（默认）
- `allkeys-lru`：删除**最少使用**的 key（最常用）
- `volatile-lru`：删除**带过期时间**的最少使用 key
- `allkeys-random`：随机删
- **企业标配**：`allkeys-lru`

### 4. 持久化（防止重启丢数据）

- **RDB**：快照，定时保存整个内存数据（快、体积小、丢数据多）
- **AOF**：日志，每条写命令记录（安全、文件大、恢复慢）
- **企业最佳实践**：**RDB + AOF 混合开启**

### 三大缓存故障：原理一句话搞懂

#### 1. 缓存穿透（最常见）

**现象**：查询**根本不存在的数据**

缓存没有 → 直接查数据库 → 数据库也没有 → 无法写入缓存

大量请求直接打穿到数据库，导致数据库崩溃

**核心**：查**不存在**的数据

#### 2. 缓存击穿（高并发）

**现象**：一个**热点 Key 过期**了

大量并发请求同时进来 → 缓存没命中 → 全部请求数据库

数据库瞬间被打挂

**核心**：**一个热点 Key** 过期，高并发访问

#### 3. 缓存雪崩（最危险）

**现象**：**大量 Key 同一时间集体过期** 或 Redis 宕机

所有请求全部打到数据库 → 数据库直接雪崩

**核心**：**大面积 Key 失效**

### 解决方案总结（背会就能面试 / 开发）

| 问题 |     核心原因      |               标准解决方案                |
| :--: | :---------------: | :---------------------------------------: |
| 穿透 |   查不存在数据    |         缓存空值、**布隆过滤器**          |
| 击穿 |   热点 Key 过期   | 互斥锁（Synchronized/Redisson）、永不过期 |
| 雪崩 | 大量 Key 同时过期 |  过期时间加随机值、Redis 集群、服务降级   |
